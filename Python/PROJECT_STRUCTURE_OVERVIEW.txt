PROJECT STRUCTURE OVERVIEW

.
├── main.py
│   - Entry point of the bot (root level)
│   - Starts market loop, 15-minute scheduler, and coordinates all engines
│   - Actively used as the main orchestrator
├── bot_state.json
│   - Persistent JSON state file for root level operations
│   - Stores current wallet, session, and thinking state
│   - Actively used as the source of truth
├── market_state.xlsx
│   - Excel workbook for market regime and status
│   - Logs timestamped market sessions and trends
│   - Actively used by ExcelManager
├── signal_analysis.xlsx
│   - Excel workbook for signal logic and indicator snapshots
│   - Logs the 'why' behind every signal or hold decision
│   - Actively used by ExcelManager
├── paper_trades.xlsx
│   - Excel workbook for simulated trade execution details
│   - Tracks entry, exit, capital used, and PnL
│   - Actively used by ExcelManager
├── risk_and_drawdown.xlsx
│   - Excel workbook for wallet health and risk metrics
│   - Tracks equity curve and drawdown percentages
│   - Actively used by ExcelManager
├── post_market_learning.xlsx
│   - Excel workbook for daily strategy feedback
│   - Stores analysis generated after market close
│   - Actively used by ExcelManager
├── Trading_Analytics.xlsx
│   - Legacy combined Excel workbook
│   - Was used before the transition to distributed workbooks
│   - Possibly redundant (now split into 5 files)
├── trading_bot_audit.db
│   - SQLite database for technical audit logs
│   - Stores signals, trades, and system events for the dashboard
│   - Actively used by EventLogger
├── replit.md
│   - Project documentation and architecture guide
│   - Contains system overview and user preferences
│   - Actively used for system context
├── pyproject.toml / uv.lock
│   - Python dependency management files
│   - Defines required packages (pandas, openpyxl, telegram, etc.)
│   - Actively used by the environment
├── Python/
│   ├── main.py
│   │   - Duplicate of root main.py or legacy version
│   │   - Contains similar orchestration logic
│   │   - Possibly redundant/duplicate of root main.py
│   ├── index.html
│   │   - Frontend for the trading dashboard
│   │   - Displays bot thinking, wallet stats, and logs
│   │   - Actively used by dashboard_api.py
│   ├── the/
│   │   ├── state_manager.py
│   │   │   - Central authority for reading/writing state
│   │   │   - Handles thread-safe JSON operations
│   │   │   - Actively used
│   │   ├── market_data_and_signal.py
│   │   │   - Generates simulated price data and trading signals
│   │   │   - Enforces session rules and candle validation
│   │   │   - Actively used
│   │   ├── trade_execution_and_mode.py
│   │   │   - Handles virtual wallet logic and paper trade execution
│   │   │   - Manages margin, leverage, and trade objects
│   │   │   - Actively used
│   │   ├── trade_management_and_risk.py
│   │   │   - Monitors positions and calculates unrealized PnL
│   │   │   - Handles Stop Loss and mandatory session exits
│   │   │   - Actively used
│   │   ├── excel_manager.py
│   │   │   - Manages all Excel workbook creation and appending
│   │   │   - Handles the 5 specialized analytics files
│   │   │   - Actively used
│   │   ├── event_logger.py
│   │   │   - Coordinates SQLite and Excel logging
│   │   │   - Bridges engine events to persistent storage
│   │   │   - Actively used
│   │   ├── session_engine.py
│   │   │   - Tracks IST market hours and session transitions
│   │   │   - Triggers post-market analysis and learning
│   │   │   - Actively used
│   │   ├── telegram_reporter.py
│   │   │   - Handles 15-minute and daily Telegram updates
│   │   │   - Manages multi-file attachments and summaries
│   │   │   - Actively used
│   │   └── dashboard_api.py
│   │       - FastAPI server for the web terminal
│   │       - Exposes state, logs, and manual report triggers
│   │       - Actively used
│   ├── bot_state.json / trading_bot_audit.db
│   │   - Nested duplicates of root state files
│   │   - Created due to execution path differences
│   │   - Possibly redundant duplicates
│   └── replit.md / pyproject.toml
│       - Duplicates of root documentation/config files
│       - Possibly redundant
└── attached_assets/
    - Contains raw prompt files and historical requirements
    - Used for documentation/reference only
